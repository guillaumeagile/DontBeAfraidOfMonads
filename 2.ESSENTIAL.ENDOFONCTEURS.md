# Endofoncteurs: Functors, Closures, and the Result Monad

## Core Concepts

### Functor
A **functor** is a container with a `map` operation that applies a function to its contents without breaking the container structure.

```csharp
// IEnumerable<T> is a functor - Select is its map
IEnumerable<int> numbers = new[] { 1, 2, 3 };
IEnumerable<int> doubled = numbers.Select(x => x * 2);
// Result: [2, 4, 6]
```

### Endofunctor
An **endofunctor** is a functor that stays within the same type system. It transforms types in your language to other types in the same language.

- **Functor**: A container with `map`
- **Endofunctor**: A container with `map` that stays in the same type system
- **Monad**: An endofunctor with `flatMap` and `pure`, organized as a monoid

In practice, **all functors you use are endofunctors**: `List`, `Optional`, `Result`, `Task`, etc.

## Part 1: Functions as First-Class Values

### Func Delegate
```csharp
// Type-safe function reference
Func<int, int> double_number = x => x * 2;
int result = double_number(5);  // 10

// Multiple parameters - last type is return type
Func<int, int, int> add = (a, b) => a + b;
int sum = add(3, 7);  // 10

// Functions as parameters
int ApplyOperation(int x, int y, Func<int, int, int> operation)
{
    return operation(x, y);
}
```

### Lambda Expressions
```csharp
// Single parameter - no parentheses needed
var square = x => x * x;

// Multiple parameters - use parentheses
var subtract = (double a, int b) => a - b;

// Multiple statements - use braces and return
Func<int, int> increment_twice = x =>
{
    int step1 = x + 1;
    int step2 = step1 + 1;
    return step2;
};
```

## Part 2: Closures and Capture

### Variable Capture
```csharp
// Lambdas capture variables by REFERENCE, not value
int multiplier = 3;
Func<int, int> multiply = x => x * multiplier;
multiplier = 4;  // Change the captured variable
int result = multiply(4);  // 16 (uses new value 4, not 3)
```

### Closure: Function Factory
```csharp
// Create specialized functions with fixed parameters
Func<int, Func<int, int>> CreateMultiplier = factor =>
{
    return x => x * factor;  // Each call captures its own 'factor'
};

var multiply_by_2 = CreateMultiplier(2);
var multiply_by_3 = CreateMultiplier(3);

multiply_by_2(10);  // 20
multiply_by_3(10);  // 30
```

### Practical: Creating Validators
```csharp
Func<int, Func<int, bool>> CreateRangeValidator = (min) =>
{
    return x => x >= min;
};

var isPositive = CreateRangeValidator(0);
var isGreaterThan10 = CreateRangeValidator(10);

isPositive(-5);      // false
isGreaterThan10(15); // true
```

## Part 3: Functors - Containers with Map

### Select Preserves Structure
```csharp
// Map applies a function inside the container without breaking it
IEnumerable<string> words = new[] { "hello", "world" };
IEnumerable<int> lengths = words.Select(w => w.Length);
// Result: [5, 5] - still enumerable

// Chaining maps
IEnumerable<int> result = numbers
    .Select(x => x * 2)      // double
    .Select(x => x + 1)      // add 1
    .Select(x => x * x);     // square
```

### Select vs SelectMany
```csharp
// Select keeps nested structure
var nested = words.Select(w => w.ToCharArray());
// Result: [['h','e','l','l','o'], ['w','o','r','l','d']]

// SelectMany flattens the structure
var flattened = words.SelectMany(w => w.ToCharArray());
// Result: ['h','e','l','l','o','w','o','r','l','d']
```

### Custom Functors
```csharp
// Any type with Select is a functor
var optional = new Optional<int>(42);
Optional<int> doubled = optional.Select(x => x * 2);
// Result: Optional<int>(84)
```

## Part 4: Result<T, E> Monad

### Success or Failure
```csharp
Result<int, Error> success = Result<int, Error>.Ok(42);
Result<int, Error> failure = Result<int, Error>.Err(Error.InvalidFormat("UserId"));

success.IsOk;   // true
failure.IsOk;   // false
```

### Chaining with Bind (SelectMany)
```csharp
Result<int, Error> ParseUserId(string input)
{
    return int.TryParse(input, out var id)
        ? Result<int, Error>.Ok(id)
        : Result<int, Error>.Err(Error.InvalidFormat("UserId"));
}

Result<string, Error> FetchUserName(int userId)
{
    return userId > 0
        ? Result<string, Error>.Ok($"User_{userId}")
        : Result<string, Error>.Err(Error.ValidationFailed("UserId", "must be positive"));
}

// Chain: parse ID, then fetch user name
var result = ParseUserId("123")
    .SelectMany(FetchUserName);

result.IsOk;     // true
result.Value;    // "User_123"
```

### Error Stops the Chain
```csharp
// If any operation fails, subsequent operations are skipped
var result = ParseUserId("invalid")
    .SelectMany(FetchUserName);

result.IsOk;                    // false
result.Error.Code;              // "INVALID_FORMAT"
result.Error.Message;           // "Invalid UserId format"
```

### LINQ Query Syntax
```csharp
// Result monad works with LINQ
var queryResult = from userId in ParseUserId("456")
                  from userName in FetchUserName(userId)
                  select userName;

// Equivalent to:
var fluentResult = ParseUserId("456")
    .SelectMany(FetchUserName);
```

### Practical: Web API Validation
```csharp
Result<string, Error> ValidateEmail(string email)
{
    return email.Contains("@")
        ? Result<string, Error>.Ok(email)
        : Result<string, Error>.Err(Error.InvalidFormat("Email"));
}

Result<int, Error> ValidateAge(int age)
{
    return age >= 18
        ? Result<int, Error>.Ok(age)
        : Result<int, Error>.Err(Error.ValidationFailed("Age", "must be at least 18"));
}

Result<(string email, int age), Error> RegisterUser(string email, int age)
{
    return ValidateEmail(email)
        .SelectMany(validEmail =>
            ValidateAge(age)
                .SelectMany(validAge =>
                    Result<(string, int), Error>.Ok((validEmail, validAge))
                )
        );
}

var success = RegisterUser("user@example.com", 25);
success.IsOk;    // true
success.Value;   // ("user@example.com", 25)
```

### Match: Catamorphism for Result
```csharp
// Pattern 1: Match returning a value
var result = ParseUserId("42");
string message = result.Match(
    onOk: id => $"Successfully parsed: {id}",
    onErr: err => $"Error: {err.Code}"
);

// Pattern 2: Match with side effects
result.Match(
    onOk: id => Console.WriteLine($"User ID: {id}"),
    onErr: err => Console.WriteLine($"Error: {err.Code}")
);

// Pattern 3: Converting to HTTP response
var apiResponse = FetchUser(123).Match(
    onOk: userName => new { success = true, data = userName, error = (string)null },
    onErr: err => new { success = false, data = (string)null, error = err.Code }
);
```

## Key Insights

1. **Functions are values**: They can be passed around, stored, and returned
2. **Closures capture context**: Function factories create specialized functions
3. **Functors preserve structure**: Map keeps the container intact
4. **Result monad handles errors**: Chain operations that might fail without explicit error handling
5. **Match is fluent error handling**: No if/else needed - reduces cyclomatic complexity

---

## Summary

- **Functor**: Container with `map` (Select)
- **Endofunctor**: Functor that stays in the same type system
- **Monad**: Endofunctor with `flatMap` (SelectMany) and `pure` (Return)
- **Result<T, E>**: Monad for error handling - chains operations that might fail
- **Match**: Catamorphism - handles both success and failure cases cleanly

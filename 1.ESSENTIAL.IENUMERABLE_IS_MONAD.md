# IEnumerable is a Monad: Essential Concepts

## Two Structures in One

`IEnumerable<T>` has **both** a monoid and a monad structure:

### Monoid (Combining Values)
A monoid combines values of the same type:

| Concept | Implementation | Example |
|---------|----------------|---------|
| **Operation** | `Concat` | `[1,2].Concat([3,4])` → `[1,2,3,4]` |
| **Identity** | `Empty` | `[].Concat([1,2])` → `[1,2]` |

**Laws:**
- Associativity: `a.Concat(b).Concat(c) == a.Concat(b.Concat(c))`
- Identity: `Empty.Concat(x) == x`

### Monad (Chaining Computations)
A monad chains operations that produce wrapped values:

| Concept | Implementation | Example |
|---------|----------------|---------|
| **Return** | `new[] { x }` | Wrap a value into `IEnumerable<T>` |
| **Bind** | `SelectMany` | Chain `IEnumerable<T> → (T → IEnumerable<U>) → IEnumerable<U>` |

**Laws:**
- Left identity: `Return(a).Bind(f) == f(a)`
- Right identity: `m.Bind(Return) == m`
- Associativity: `m.Bind(f).Bind(g) == m.Bind(x => f(x).Bind(g))`

## How They Relate

**Bind = Map + Flatten**

```csharp
// Step 1: Map gives M<M<U>>
IEnumerable<IEnumerable<int>> mapped = source.Select(Neighbours);

// Step 2: Flatten using Concat (the monoid)
IEnumerable<int> flattened = mapped.Aggregate(
    Enumerable.Empty<int>(),
    (acc, seq) => acc.Concat(seq));

// This is equivalent to Bind:
source.SelectMany(Neighbours);
```

The monoid is the "how" behind Bind's flatten step. Without Concat, you couldn't flatten.

## Return: Wrapping a Value

```csharp
// Wrap a single value
IEnumerable<int> wrapped = new[] { 42 };

// Or with LINQ
IEnumerable<int> lifted = from _ in new[] { 0 } select 42;
```

## Bind: Chaining Computations

### Simple Case: Single Result Per Element
```csharp
IEnumerable<int> source = new[] { 1, 2, 3 };
IEnumerable<int> Increment(int x) => new[] { x + 1 };

// Using SelectMany (bind)
IEnumerable<int> result = source.SelectMany(Increment);

// Using LINQ query syntax
var query = from x in source
            from i in Increment(x)
            select i;

// Result: [2, 3, 4]
```

### Expansion Case: Multiple Results Per Element
```csharp
IEnumerable<int> Neighbours(int x) => new[] { x - 1, x + 1 };

IEnumerable<int> result = source.SelectMany(Neighbours);
// 1 → [0, 2]
// 2 → [1, 3]
// 3 → [2, 4]
// Result: [0, 2, 1, 3, 2, 4]  (note: 2 appears twice)
```

## Monoid in Action: Aggregate

Aggregate folds a sequence using the monoid operation:

```csharp
IEnumerable<string> article1Tags = new[] { "c#", "monad" };
IEnumerable<string> article2Tags = new[] { "linq", "monad" };
IEnumerable<string> article3Tags = new[] { "c#", "linq", "fun" };

IEnumerable<IEnumerable<string>> allTags = new[] { article1Tags, article2Tags, article3Tags };

// Flatten using Aggregate + Concat (the monoid)
IEnumerable<string> collected = allTags.Aggregate(
    Enumerable.Empty<string>(),
    (acc, tags) => acc.Concat(tags));

// Result: ["c#", "monad", "linq", "monad", "c#", "linq", "fun"]

// Equivalent to:
allTags.SelectMany(x => x);
```

## The Functional Pipeline: Map → Filter → Reduce

```csharp
IEnumerable<int> source = Enumerable.Range(1, 20);

int result = source
    .Select(x => x * 3)              // Map: transform
    .Where(x => x % 2 == 0)          // Filter: keep only
    .Aggregate(0, (acc, x) => acc + x); // Reduce: collapse

// LINQ query syntax (Map + Filter):
var query = from x in source
            let tripled = x * 3
            where tripled % 2 == 0
            select tripled;

int resultQuery = query.Aggregate(0, (acc, x) => acc + x);
```

## LINQ Query Syntax = Syntactic Sugar for Bind

The compiler translates LINQ queries into method calls:

```csharp
// Query syntax
var query = from x in new[] { 1, 2, 3 }
            from y in new[] { x, x * 10 }
            select $"{x}:{y}";

// Desugared (what the compiler actually runs)
IEnumerable<string> desugared = new[] { 1, 2, 3 }
    .SelectMany(x => new[] { x, x * 10 }, (x, y) => $"{x}:{y}");
```

**Every `from` clause after the first is a `SelectMany` (bind) call.**

---

## Key Insight

`IEnumerable<T>` is a "monad plus":
- The **monad** (Return + Bind) lets you chain computations
- The **monoid** (Empty + Concat) lets you combine results
- Bind uses the monoid internally to flatten

This is why LINQ is so powerful: it's monadic composition with the monoid built-in.

## Dictionary: FP Concepts → C# LINQ

| FP Concept | C# / LINQ | What It Does |
|-----------|----------|-------------|
| Return / pure | `new[] { x }` | Wrap a value |
| Bind / >>= | `SelectMany` | Chain computations |
| fmap / Map | `Select` | Transform inside |
| Join / Flatten | `.SelectMany(x => x)` | Collapse nesting |
| Guard / Filter | `Where` | Keep matching |
| mzero / Empty | `Enumerable.Empty<T>()` | Identity element |
| mplus / <> | `Concat` | Combine two |
| fold / reduce | `Aggregate` | Collapse to one value |

---

## Conclusion

`IEnumerable<T>` is a monad because it has:
1. **Return**: Wraps values
2. **Bind**: Chains computations without nesting

It's also a monoid because it has:
1. **Concat**: Combines sequences
2. **Empty**: Identity element

Together, they make LINQ a powerful functional programming tool you use every day.
